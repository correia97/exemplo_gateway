## Tasks

### Task 1: Create init-routes.sh script
Create `.planning/phases/05-apisix-gateway/init-routes.sh` — the init script that configures Kong via Admin API at container startup.

**Key design decisions:**
- The script runs inside the Kong container, so Admin API is at `http://127.0.0.1:9180`
- All configurable values come from environment variables with bash defaults
- Only GET routes are created here (no auth needed). Write routes come in 05-03.
- CORS plugin is added to GET routes from the start (browsers need CORS on all requests including GET)
- `strip_path: true` on both routes so upstream gets clean paths (`/characters` not `/api/dragonball/characters`)

```bash
#!/bin/bash
# init-routes.sh — Configure Kong routes via Admin API
set -e

# === Configuration from environment (with defaults for dev) ===
ADMIN_API="http://127.0.0.1:9180/apisix/admin"
ADMIN_KEY="${ADMIN_KEY:-edd1c9f034335f136f87ad84b625c8f1}"

UPSTREAM_DRAGONBALL="${UPSTREAM_DRAGONBALL_API:-http://192.168.3.4:5000}"
UPSTREAM_MUSIC="${UPSTREAM_MUSIC_API:-http://192.168.3.4:5002}"

CORS_ORIGINS="${CORS_ORIGINS:-http://localhost:5173,http://localhost:3000,http://localhost,http://localhost:5003}"

# === Wait for Admin API ===
echo "Waiting for Kong Admin API..."
until curl -s -o /dev/null -w "%{http_code}" "$ADMIN_API/routes" \
    -H "X-API-KEY: $ADMIN_KEY" | grep -q "200"; do
    sleep 2
done
echo "Admin API ready."

# === Create Upstreams ===

echo "Creating upstream: dragonball → $UPSTREAM_DRAGONBALL"
curl -s -X PUT "$ADMIN_API/upstreams/dragonball" \
    -H "X-API-KEY: $ADMIN_KEY" \
    -H "Content-Type: application/json" \
    -d "$(cat <<EOF
{
    "type": "roundrobin",
    "nodes": {
        "$UPSTREAM_DRAGONBALL": 1
    }
}
EOF
)"

echo "Creating upstream: music → $UPSTREAM_MUSIC"
curl -s -X PUT "$ADMIN_API/upstreams/music" \
    -H "X-API-KEY: $ADMIN_KEY" \
    -H "Content-Type: application/json" \
    -d "$(cat <<EOF
{
    "type": "roundrobin",
    "nodes": {
        "$UPSTREAM_MUSIC": 1
    }
}
EOF
)"

# === Create GET Routes (no auth — public reads) ===

echo "Creating GET route: /api/dragonball/*"
curl -s -X PUT "$ADMIN_API/routes/dragonball-get" \
    -H "X-API-KEY: $ADMIN_KEY" \
    -H "Content-Type: application/json" \
    -d "$(cat <<EOF
{
    "uri": "/api/dragonball/*",
    "methods": ["GET"],
    "strip_path": true,
    "upstream_id": "dragonball",
    "plugins": {
        "cors": {
            "allow_origins": "$CORS_ORIGINS",
            "allow_methods": "GET,POST,PUT,DELETE,OPTIONS",
            "allow_headers": "*",
            "expose_headers": "X-Correlation-Id",
            "allow_credential": true,
            "max_age": 86400
        }
    }
}
EOF
)"

echo "Creating GET route: /api/music/*"
curl -s -X PUT "$ADMIN_API/routes/music-get" \
    -H "X-API-KEY: $ADMIN_KEY" \
    -H "Content-Type: application/json" \
    -d "$(cat <<EOF
{
    "uri": "/api/music/*",
    "methods": ["GET"],
    "strip_path": true,
    "upstream_id": "music",
    "plugins": {
        "cors": {
            "allow_origins": "$CORS_ORIGINS",
            "allow_methods": "GET,POST,PUT,DELETE,OPTIONS",
            "allow_headers": "*",
            "expose_headers": "X-Correlation-Id",
            "allow_credential": true,
            "max_age": 86400
        }
    }
}
EOF
)"

echo "Route initialization complete."
```

### Task 2: Wire init-routes.sh into Kong container
Update `src/OpenCode.AppHost/Program.cs` so that the init script runs after Kong starts. The approach: override the Kong container's `ENTRYPOINT` to run the init script first, then start Kong.

Find the existing Kong container definition and modify it:

Before:
```csharp
var apisix = builder.AddContainer("apisix", "apache/apisix:3.9.1-alpine")
    .WithEnvironment("UPSTREAM_DRAGONBALL_API", "http://host.docker.internal:5000")
    .WithEnvironment("UPSTREAM_MUSIC_API", "http://host.docker.internal:5002")
    .WithEnvironment("KEYCLOAK_URL", "http://host.docker.internal:8080")
    .WithEnvironment("CLIENT_SECRET_DRAGONBALL", "dragonball-secret")
    .WithEnvironment("CLIENT_SECRET_MUSIC", "music-secret")
    .WithEnvironment("ADMIN_KEY", "edd1c9f034335f136f87ad84b625c8f1")
    .WithBindMount(apisixConfigPath, "/usr/local/apisix/conf")
    .WithEndpoint(port: 8000, targetPort: 9080, scheme: "http", name: "proxy")
    .WithEndpoint(port: 9180, targetPort: 9180, scheme: "http", name: "admin")
    .WaitFor(etcd);
```

After:
```csharp
var apisixScriptsPath = Path.Combine(apisixConfigPath, ".."); // points to .planning/phases/05-apisix-gateway/

var apisix = builder.AddContainer("apisix", "apache/apisix:3.9.1-alpine")
    .WithEnvironment("UPSTREAM_DRAGONBALL_API", "http://host.docker.internal:5000")
    .WithEnvironment("UPSTREAM_MUSIC_API", "http://host.docker.internal:5002")
    .WithEnvironment("KEYCLOAK_URL", "http://host.docker.internal:8080")
    .WithEnvironment("CLIENT_SECRET_DRAGONBALL", "dragonball-secret")
    .WithEnvironment("CLIENT_SECRET_MUSIC", "music-secret")
    .WithEnvironment("ADMIN_KEY", "edd1c9f034335f136f87ad84b625c8f1")
    .WithBindMount(apisixConfigPath, "/usr/local/apisix/conf")
    .WithBindMount(apisixScriptsPath, "/scripts")  // NEW — mount init-routes.sh
    .WithEndpoint(port: 8000, targetPort: 9080, scheme: "http", name: "proxy")
    .WithEndpoint(port: 9180, targetPort: 9180, scheme: "http", name: "admin")
    .WithEntrypoint("/bin/sh", "-c", "chmod +x /scripts/init-routes.sh && /scripts/init-routes.sh && exec /usr/local/openresty/bin/openresty -p /usr/local/apisix -g 'daemon off;'")
    .WaitFor(etcd);
```

**Why this entrypoint:**
- The default Kong container entrypoint starts OpenResty (which runs Kong). We override it to:
  1. Make the init script executable
  2. Run the init script (creates upstreams and routes)
  3. `exec` the original OpenResty command to start Kong as PID 1
- The `exec` is critical: it replaces the shell process so signals (SIGTERM, etc.) go to OpenResty directly
- The original entrypoint path is discovered from the Kong Dockerfile (`/usr/local/openresty/bin/openresty -p /usr/local/apisix -g 'daemon off;'` — verified for apache/apisix:3.9.1-alpine)
