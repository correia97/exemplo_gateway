# 05-01: APISIX Container + etcd in Aspire

## Objective
Add etcd (APISIX config store) and APISIX containers to the Aspire AppHost, with well-known port bindings for the two API projects so APISIX can route to them.

**Purpose:** Create the infrastructure that hosts the gateway. Without this plan, no routing can occur.

**Output:** `config.yaml`, updated `Program.cs` with etcd + APISIX containers, and API projects bound to known ports.

## Tasks

### Task 1: Add well-known ports to API projects in AppHost
Update `src/OpenCode.AppHost/Program.cs` to bind the API projects to well-known ports so APISIX can route to them via `host.docker.internal`.

### Task 2: Add etcd container
After the keycloak block, add etcd. et  does not require authentication for the PoC.

**Important:** The project has a rule against Bitnami images. Use `quay.io/coreos/etcd:v3.5`.

```csharp
var etcd = builder.AddContainer("etcd", "quay.io/coreos/etcd:v3.5")
    .WithEnvironment("ALLOW_NONE_AUTHENTICATION", "yes")
    .WithEnvironment("ETCD_LISTEN_CLIENT_URLS", "http://0.0.0.0:2379")
    .WithEnvironment("ETCD_ADVERTISE_CLIENT_URLS", "http://etcd:2379")
    .WithEndpoint(port: 2379, targetPort: 2379, scheme: "http", name: "client")
    .WithEndpoint(port: 2380, targetPort: 2380, scheme: "http", name: "peer");
```

### Task 3: Add APISIX container
Add APISIX container after the etcd block. APISIX connects to etcd at startup and caches configuration.

```csharp
var apisixConfigPath = Path.Combine(
    Directory.GetParent(typeof(Program).Assembly.Location)!.Parent!.Parent!.Parent!.Parent!.FullName,
    "..", ".planning", "phases", "05-apisix-gateway");

var apisix = builder.AddContainer("apisix", "apache/apisix:3.9.1-alpine")
    .WithEnvironment("UPSTREAM_DRAGONBALL_API", "http://host.docker.internal:5000")
    .WithEnvironment("UPSTREAM_MUSIC_API", "http://host.docker.internal:5002")
    .WithEnvironment("KEYCLOAK_URL", "http://host.docker.internal:8080")
    .WithEnvironment("CLIENT_SECRET_DRAGONBALL", "dragonball-secret")
    .WithEnvironment("CLIENT_SECRET_MUSIC", "music-secret")
    .WithEnvironment("ADMIN_KEY", "edd1c9f034335f136f87ad84b625c8f1")
    .WithBindMount(apisixConfigPath, "/usr/local/apisix/conf")
    .WithEndpoint(port: 8000, targetPort: 9080, scheme: "http", name: "proxy")
    .WithEndpoint(port: 9180, targetPort: 9180, scheme: "http", name: "admin")
    .WaitFor(etcd);
```

**Port mapping rationale:**
- Port 8000 → 9080 (APISIX proxy — external entry point per GATE-01)
- Port 9180 → 9180 (APISIX Admin API — internal for init script)

### Task 4: Create initial config.yaml
Create `.planning/phases/05-apisix-gateway/config.yaml` with minimal APISIX configuration. The plugin whitelist is critical — without it, the Admin API rejects plugin configurations.

```yaml
apisix:
  node_listen: 9080
  enable_admin: true
  enable_dev_mode: false
  proxy_cache: false
  plugins:
    - openid-connect
    - cors
    - request-id
    - prometheus

nginx_config:
  error_log_level: warn

deployment:
  admin:
    admin_key:
      - name: admin
        key: edd1c9f034335f136f87ad84b625c8f1
        role: admin
    allow_admin:
      - 0.0.0.0/0
    admin_listen:
      ip: 0.0.0.0
      port: 9180
  etcd:
    host:
      - "http://etcd:2379"
    prefix: "/apisix"
    timeout: 30
```

**Why this config:**
- `plugins` whitelist — APISIX 3.x requires explicit plugin enablement. Without listing them, Admin API returns "invalid plugin name" for openid-connect, cors, etc.
- `etcd.host` — points to container service name `etcd` on port 2379
- `admin_key` — will be overridable via environment variable in a later plan; using default for now per CONTEXT.md
- `allow_admin: 0.0.0.0/0` — allows Admin API access from any container IP (safe in dev, init script calls from within the container)
