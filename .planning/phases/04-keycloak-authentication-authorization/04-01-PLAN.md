# 04-01: Keycloak Container + PostgreSQL in Aspire

## Objective
Add PostgreSQL container to Aspire AppHost (needed for both APIs and Keycloak), then add Keycloak container with PostgreSQL backend connection.

## Tasks

### Task 1: Add PostgreSQL container to AppHost
1. Add `Aspire.Hosting.PostgreSQL` package version to `Directory.Packages.props`
2. Add `PackageReference` to `src/OpenCode.AppHost/OpenCode.AppHost.csproj`
3. Update `src/OpenCode.AppHost/Program.cs`:
   - Add `builder.AddPostgres("postgres")` container
   - Mount `src/OpenCode.Domain/Data/init.sql` to `/docker-entrypoint-initdb.d/`
   - Add `AddDatabase("dragonball")` and `AddDatabase("music")` resources
   - Wire connection strings to API projects via `WithReference`
4. Verify: `dotnet build` passes

### Task 2: Add Keycloak container
1. Update `src/OpenCode.AppHost/Program.cs`:
   - Add raw Keycloak container: `builder.AddContainer("keycloak", "quay.io/keycloak/keycloak:latest")`
   - Set environment variables: `KC_DB=postgres`, `KC_DB_URL`, `KC_DB_USERNAME=keycloak_user`, `KC_DB_PASSWORD=keycloak_pass`, `KC_IMPORT_REALM=opencode-realm.json`
   - Mount realm JSON directory: `WithBindMount("../phases/04-keycloak-authentication-authorization", "/opt/keycloak/data/import")`
   - Set `KEYCLOAK_ADMIN=admin`, `KEYCLOAK_ADMIN_PASSWORD=admin`
   - Link to PostgreSQL: `WithReference(pgResource)`
2. Verify: `dotnet build` passes

### Task 3: Add .gitignore and commit
1. Verify `.gitignore` exists (bin/, obj/ excluded)
2. Commit each task atomically

## Success Criteria
- [ ] `dotnet build` passes with 0 errors
- [ ] PostgreSQL container configured with init.sql mount
- [ ] Keycloak container configured with PG backend and realm import mount
- [ ] Connection strings wired to both API projects via Aspire references
