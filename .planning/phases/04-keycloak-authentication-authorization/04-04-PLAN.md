# 04-04: Verify Auth Flow End-to-End

## Objective
Verify the complete authentication and authorization flow: public GET access, protected mutations, role-based access control, and token validation.

## Tasks

### Task 1: Verify build and container readiness
1. `dotnet build` passes
2. AppHost starts successfully (skip if containers unavailable in dev)
3. Verify Keycloak realm JSON is syntactically valid JSON

### Task 2: Verify realm JSON structure
1. Check realm name is `opencode`
2. Check roles: `viewer`, `editor`
3. Check clients: `dragonball-api`, `music-api`, `frontend`
4. Check users: `viewer1`, `editor1` with correct roles
5. Check access token lifespan is 5 minutes

### Task 3: Verify auth flow documentation
Document the expected behavior in SUMMARY.md:

| Scenario | Endpoint | Token | Expected Status |
|----------|----------|-------|-----------------|
| Public GET | GET /api/characters | None | 200 OK |
| Unauthenticated POST | POST /api/characters | None | 401 Unauthorized |
| Viewer denied POST | POST /api/characters | viewer1 | 403 Forbidden |
| Editor allowed POST | POST /api/characters | editor1 | 201 Created |
| Editor PUT | PUT /api/characters/1 | editor1 | 200 OK |
| Editor DELETE | DELETE /api/characters/1 | editor1 | 204 No Content |
| Same for Music API | POST /api/genres | editor1 | 201 Created |
| Same for Music API | GET /api/artists | None | 200 OK |

### Task 4: Write verification summary
Create SUMMARY.md documenting the auth flow configuration and expected behavior.

## Success Criteria
- [ ] All auth scenarios documented
- [ ] Realm JSON validated syntactically
- [ ] Code compiles with 0 errors
- [ ] Auth flow documented clearly in SUMMARY.md
