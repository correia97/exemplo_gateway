# 04-02: Keycloak Realm Configuration

## Objective
Create the Keycloak realm JSON configuration with OIDC clients, roles, and test users for the `opencode` realm.

## Tasks

### Task 1: Create opencode-realm.json
Create `.planning/phases/04-keycloak-authentication-authorization/opencode-realm.json` with:
- **Realm**: `opencode`, enabled
- **Realm roles**: `viewer`, `editor`
- **OIDC clients**:
  - `dragonball-api`: confidential, secret `dragonball-secret`, service accounts enabled, direct access grants
  - `music-api`: confidential, secret `music-secret`, service accounts enabled, direct access grants
  - `frontend`: public, standard flow enabled, redirect URIs `http://localhost:5173/*`
- **Protocol mapper** on each API client: flatten `realm_access.roles` into user attribute/claim
- **Users**:
  - `viewer1` / `viewer1`: realm role `viewer`
  - `editor1` / `editor1`: realm roles `viewer`, `editor`
- Access token lifespan: 5 minutes
- Refresh token: enabled

### Task 2: Validate JSON
1. Verify JSON is valid (no trailing commas, valid structure)
2. Verify realm import works with KC_IMPORT_REALM env var

## Success Criteria
- [ ] Valid realm JSON file created
- [ ] Realm named `opencode` with both roles
- [ ] 3 OIDC clients (2 confidential, 1 public)
- [ ] 2 test users with correct role assignments
- [ ] 5-minute access token lifespan configured
