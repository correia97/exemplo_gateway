Modify `CorrelationIdMiddleware` class to inject `ILogger<CorrelationIdMiddleware>` and wrap the `await _next(context)` call in an `ILogger.BeginScope`:

1. **Add ILogger constructor parameter** — inject via constructor. The middleware is resolved from DI, so ILogger<T> is available.

2. **Create scope around next()** — wrap `await _next(context)` in `using var _ = _logger.BeginScope(...)` that includes the CorrelationId.

Full updated class:

```csharp
internal class CorrelationIdMiddleware
{
    private readonly RequestDelegate _next;
    private readonly ILogger<CorrelationIdMiddleware> _logger;
    private const string CorrelationIdHeader = "X-Correlation-Id";

    public CorrelationIdMiddleware(RequestDelegate next, ILogger<CorrelationIdMiddleware> logger)
    {
        _next = next;
        _logger = logger;
    }

    public async Task InvokeAsync(HttpContext context)
    {
        var correlationId = context.Request.Headers[CorrelationIdHeader].FirstOrDefault()
                            ?? Guid.NewGuid().ToString();

        context.Response.OnStarting(() =>
        {
            context.Response.Headers[CorrelationIdHeader] = correlationId;
            return Task.CompletedTask;
        });

        context.TraceIdentifier = correlationId;

        using (_logger.BeginScope(new { CorrelationId = correlationId }))
        {
            await _next(context);
        }
    }
}
```

Key details:
- The `using` block ensures the scope is disposed after the response completes — automatic cleanup
- Using anonymous object `new { CorrelationId = correlationId }` creates a structured log property named `CorrelationId` — no custom scope type needed
- The `Microsoft.Extensions.Logging.ILogger<T>` is automatically registered in DI by `builder.Logging.AddOpenTelemetry()` which is already called
- **No additional using directives needed** — `Microsoft.Extensions.Logging` is already imported via `using Microsoft.Extensions.Logging;` at the top (line 5)
- This works with all ILogger calls throughout the app — `_logger.LogInformation("...")` in endpoints, middleware, services will all include `CorrelationId` in their scope
- The existing `OtlpLogExporter` (configured by `AddOpenTelemetry()` on logging) will export the CorrelationId scope property to Jaeger as log attributes

Per D-04: Minimal change — ~5 lines added to existing middleware, zero changes needed in endpoint code.
