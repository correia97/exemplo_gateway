<!--
MSBuild Central Package Management Schema:

Directory.Packages.props structure:
```xml
<Project>
  <PropertyGroup>
    <ManagePackageVersionsCentrally>true</ManagePackageVersionsCentrally>
  </PropertyGroup>
  <ItemGroup>
    <PackageVersion Include="PackageName" Version="1.2.3" />
  </ItemGroup>
</Project>
```

csproj without Version (using central management):
```xml
<PackageReference Include="PackageName" />   <!-- no Version attribute -->
```

When MSBuild finds `<PackageReference>` without `Version`, it looks up `Directory.Packages.props` for a matching `<PackageVersion>`.

Required: `Directory.Packages.props` must be at or above the csproj in the directory hierarchy.
The solution root (`D:\projetos\opencodeDeepseek\`) is where it lives.

Template csproj files may use floating versions like `Version="9.0.0-*"` or specific versions like `Version="8.2.0"`.
-->
</interfaces>
