## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP Request Entry → Service | Incoming HTTP requests cross from external client into the service pipeline |
| Service → OTLP Exporter | OpenTelemetry data flows from service to Aspire Dashboard (OTLP) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04-01 | S (Spoofing) | Correlation ID header | accept | Correlation ID is for traceability, not authentication. Client can send any X-Correlation-Id value; this is expected behavior for distributed tracing. |
| T-01-04-02 | I (Information Disclosure) | OTLP exporter | accept | Aspire Dashboard is development-only. Production OTLP configuration is handled in Phase 8 (production deployment). |
| T-01-04-03 | D (Denial of Service) | CorrelationIdMiddleware | mitigate | Middleware uses `Guid.NewGuid()` as fallback — safe. No unbounded allocations. TraceIdentifier is a string property on HttpContext, no DoS vector. |
</threat_model>
