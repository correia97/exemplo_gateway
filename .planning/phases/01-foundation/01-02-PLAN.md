    builder.AddProject<Projects.OpenCode_DragonBall_Api>("dragonball-api")
           .WithReplicas(1);

    builder.AddProject<Projects.OpenCode_Music_Api>("music-api")
           .WithReplicas(1);

    2. Check the AppHost csproj for `<ProjectReference>` items. The aspire-apphost template typically does NOT include project references — instead, it relies on the solution structure and Aspire's build integration. If no ProjectReference exists, add them:

    ```xml
    <ItemGroup>
      <ProjectReference Include="..\OpenCode.DragonBall.Api\OpenCode.DragonBall.Api.csproj" />
      <ProjectReference Include="..\OpenCode.Music.Api\OpenCode.Music.Api.csproj" />
    </ItemGroup>
    ```

    The ProjectReference elements enable the `Projects` source generator to create the strongly-typed `Projects.OpenCode_DragonBall_Api` entries used in Program.cs.

    - `Aspire.Hosting.AppHost` (from template, version may be floating)
