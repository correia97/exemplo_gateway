<action>
Create `deploy/apisix/init-routes.sh` — adapted from `.planning/phases/05-apisix-gateway/init-routes.sh` for Docker Compose mode. The key changes:

1. **Upstream URLs now use container DNS** (not host.docker.internal). The init script reads environment variables `UPSTREAM_DRAGONBALL_API` and `UPSTREAM_MUSIC_API`. In Docker Compose mode, these are set to `http://dragonball-api:8080` and `http://music-api:8080`.

2. **Init script runs AFTER APISIX starts.** In Docker Compose mode, the entrypoint is:
   ```bash
   apisix start && sleep 3 && /scripts/init-routes.sh && apisix stop && exec /usr/local/openresty/bin/openresty -p /usr/local/apisix -g 'daemon off;'
   ```
   This starts APISIX in daemon mode, waits for the Admin API to become available, runs the init script, stops the daemon, then starts openresty in foreground (proper Docker PID 1 pattern).

3. **Remove OTEL init call** from the entrypoint chain (Phase 6 OTEL is Aspire-specific; Docker Compose mode keeps it minimal unless Jaeger is explicitly added later).

The script body (adapted from the Phase 5 version):

```bash
#!/bin/bash
set -e

ADMIN_API="http://127.0.0.1:9180/apisix/admin"
ADMIN_KEY="${ADMIN_KEY:-edd1c9f034335f136f87ad84b625c8f1}"

UPSTREAM_DRAGONBALL="${UPSTREAM_DRAGONBALL_API:-http://dragonball-api:8080}"
UPSTREAM_MUSIC="${UPSTREAM_MUSIC_API:-http://music-api:8080}"

CORS_ORIGINS="${CORS_ORIGINS:-http://localhost:5173,http://localhost:3000,http://localhost,http://localhost:5003}"

echo "Waiting for APISIX Admin API..."
until curl -s -o /dev/null -w "%{http_code}" "$ADMIN_API/routes" \
    -H "X-API-KEY: $ADMIN_KEY" | grep -q "200"; do
    sleep 2
done
echo "Admin API ready."

echo "Creating global rule: request-id (X-Correlation-Id)"
curl -s -X PUT "$ADMIN_API/global_rules/correlation-id" \
    -H "X-API-KEY: $ADMIN_KEY" \
    -H "Content-Type: application/json" \
    -d "$(cat <<EOF
{
    "plugins": {
        "request-id": {
            "header_name": "X-Correlation-Id",
            "include_in_response": true,
            "algorithm": "uuid"
        }
    }
}
EOF
)"

echo "Creating upstream: dragonball -> $UPSTREAM_DRAGONBALL"
curl -s -X PUT "$ADMIN_API/upstreams/dragonball" \
    -H "X-API-KEY: $ADMIN_KEY" \
    -H "Content-Type: application/json" \
    -d "$(cat <<EOF
{
    "type": "roundrobin",
    "nodes": {
        "$UPSTREAM_DRAGONBALL": 1
    }
}
EOF
)"

echo "Creating upstream: music -> $UPSTREAM_MUSIC"
curl -s -X PUT "$ADMIN_API/upstreams/music" \
    -H "X-API-KEY: $ADMIN_KEY" \
    -H "Content-Type: application/json" \
    -d "$(cat <<EOF
{
    "type": "roundrobin",
    "nodes": {
        "$UPSTREAM_MUSIC": 1
    }
}
EOF
)"

echo "Creating GET route: /api/dragonball/*"
curl -s -X PUT "$ADMIN_API/routes/dragonball-get" \
    -H "X-API-KEY: $ADMIN_KEY" \
    -H "Content-Type: application/json" \
    -d "$(cat <<EOF
{
    "uri": "/api/dragonball/*",
    "methods": ["GET"],
    "strip_path": true,
    "upstream_id": "dragonball",
    "plugins": {
        "cors": {
            "allow_origins": "$CORS_ORIGINS",
            "allow_methods": "GET,POST,PUT,DELETE,OPTIONS",
            "allow_headers": "*",
            "expose_headers": "X-Correlation-Id",
            "allow_credential": true,
            "max_age": 86400
        }
    }
}
EOF
)"

echo "Creating GET route: /api/music/*"
curl -s -X PUT "$ADMIN_API/routes/music-get" \
    -H "X-API-KEY: $ADMIN_KEY" \
    -H "Content-Type: application/json" \
    -d "$(cat <<EOF
{
    "uri": "/api/music/*",
    "methods": ["GET"],
    "strip_path": true,
    "upstream_id": "music",
    "plugins": {
        "cors": {
            "allow_origins": "$CORS_ORIGINS",
            "allow_methods": "GET,POST,PUT,DELETE,OPTIONS",
            "allow_headers": "*",
            "expose_headers": "X-Correlation-Id",
            "allow_credential": true,
            "max_age": 86400
        }
    }
}
EOF
)"

echo "Creating write route: /api/dragonball/* (OIDC)"
curl -s -X PUT "$ADMIN_API/routes/dragonball-write" \
    -H "X-API-KEY: $ADMIN_KEY" \
    -H "Content-Type: application/json" \
    -d "$(cat <<EOF
{
    "uri": "/api/dragonball/*",
    "methods": ["POST", "PUT", "DELETE"],
    "strip_path": true,
    "upstream_id": "dragonball",
    "plugins": {
        "cors": {
            "allow_origins": "$CORS_ORIGINS",
            "allow_methods": "GET,POST,PUT,DELETE,OPTIONS",
            "allow_headers": "*",
            "expose_headers": "X-Correlation-Id",
            "allow_credential": true,
            "max_age": 86400
        },
        "openid-connect": {
            "client_id": "dragonball-api",
            "client_secret": "${CLIENT_SECRET_DRAGONBALL:-dragonball-secret}",
            "discovery": "http://keycloak:8080/realms/opencode/.well-known/openid-configuration",
            "bearer_only": true,
            "ssl_verify": false,
            "realm": "opencode"
        }
    }
}
EOF
)"

echo "Creating write route: /api/music/* (OIDC)"
curl -s -X PUT "$ADMIN_API/routes/music-write" \
    -H "X-API-KEY: $ADMIN_KEY" \
    -H "Content-Type: application/json" \
    -d "$(cat <<EOF
{
    "uri": "/api/music/*",
    "methods": ["POST", "PUT", "DELETE"],
    "strip_path": true,
    "upstream_id": "music",
    "plugins": {
        "cors": {
            "allow_origins": "$CORS_ORIGINS",
            "allow_methods": "GET,POST,PUT,DELETE,OPTIONS",
            "allow_headers": "*",
            "expose_headers": "X-Correlation-Id",
            "allow_credential": true,
            "max_age": 86400
        },
        "openid-connect": {
            "client_id": "music-api",
            "client_secret": "${CLIENT_SECRET_MUSIC:-music-secret}",
            "discovery": "http://keycloak:8080/realms/opencode/.well-known/openid-configuration",
            "bearer_only": true,
            "ssl_verify": false,
            "realm": "opencode"
        }
    }
}
EOF
)"

echo "Route initialization complete."
```

**Design decisions:**

1. **Image tags:**
   - `postgres:17` — official PostgreSQL 17, meets INFRA-05 (no Bitnami)
   - `quay.io/keycloak/keycloak:latest` — official Keycloak image from Red Hat's registry
   - `quay.io/coreos/etcd:v3.5` — official etcd image (not Bitnami)
   - `apache/apisix:3.9.1-alpine` — official APISIX image

2. **Build context for .NET APIs:**
   - `context: .` (project root) because Dockerfiles need access to the solution structure to restore packages
   - `dockerfile: src/OpenCode.DragonBall.Api/Dockerfile` — per-project Dockerfile
   - Same pattern for Music API

3. **Frontend build context:**
   - `context: src/OpenCode.Frontend` — Dockerfile inside the frontend directory

4. **APISIX entrypoint pattern:**
   - `apisix start` — starts openresty in daemon mode
   - `sleep 3` — brief wait for Admin API availability
   - `/scripts/init-routes.sh` — creates all routes via Admin API
   - `apisix stop` — cleanly stops daemon
   - `exec /usr/local/openresty/bin/openresty ... -g 'daemon off;'` — starts in foreground as PID 1

5. **Container DNS:**
   - Postgres reachable as `postgres:5432` from other containers
   - Keycloak reachable as `keycloak:8080`
   - APISIX upstreams use `dragonball-api:8080` and `music-api:8080` (container ports, not mapped ports)

6. **Port 8080 for .NET APIs:**
   - .NET containers expose 8080 (standard ASP.NET container port) set via `ASPNETCORE_URLS=http://+:8080`
   - No host port mapping needed — services communicate over the internal network

7. **No restart on .NET APIs yet:**
   - `restart: unless-stopped` is set on all services
   - Health checks will be added in Plan 08-02 to gate depends_on properly

**Do NOT use Bitnami images.** All images must be official: mcr.microsoft.com, quay.io, postgres official, apache official, node official (for frontend build), nginx official (for frontend runtime).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Host → PostgreSQL (:5432) | Direct DB access from host — exposed for tooling but should be locked in production |
| Host → APISIX Admin API (:9180) | APISIX admin API exposed on host — can modify all routes |
| Container network | All services communicate over opencode-net — internal network, trusted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Information Disclosure | PostgreSQL :5432 exposed | accept | PoC scope; production would restrict to internal network only |
