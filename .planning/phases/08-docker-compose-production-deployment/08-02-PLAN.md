Create production-grade Dockerfiles for the two .NET APIs (DragonBall, Music) and the React frontend, plus add health checks and full environment configuration to the docker-compose.yml created in Plan 08-01. Create two Dockerfiles for the .NET 10 API projects using multi-stage builds with official `mcr.microsoft.com/dotnet` images. Both Dockerfiles follow the same pattern since both are .NET Minimal API projects with identical dependency structures.

**src/OpenCode.DragonBall.Api/Dockerfile:**

```dockerfile
# Stage 1: Build
FROM mcr.microsoft.com/dotnet/sdk:10.0 AS build
WORKDIR /src

# Copy project files for restore layer caching
COPY Directory.Packages.props .
COPY OpenCode.slnx .
COPY src/OpenCode.Domain/OpenCode.Domain.csproj src/OpenCode.Domain/
COPY src/OpenCode.ServiceDefaults/OpenCode.ServiceDefaults.csproj src/OpenCode.ServiceDefaults/
COPY src/OpenCode.DragonBall.Api/OpenCode.DragonBall.Api.csproj src/OpenCode.DragonBall.Api/
RUN dotnet restore src/OpenCode.DragonBall.Api/OpenCode.DragonBall.Api.csproj

# Copy source and publish
COPY src/OpenCode.Domain/ src/OpenCode.Domain/
COPY src/OpenCode.ServiceDefaults/ src/OpenCode.ServiceDefaults/
COPY src/OpenCode.DragonBall.Api/ src/OpenCode.DragonBall.Api/
RUN dotnet publish src/OpenCode.DragonBall.Api/OpenCode.DragonBall.Api.csproj \
    -c Release \
    -o /app \
    --no-restore

# Stage 2: Runtime
FROM mcr.microsoft.com/dotnet/aspnet:10.0 AS runtime
WORKDIR /app

# Create non-root user (Ubuntu-based aspnet image)
RUN addgroup --system --gid 1000 appgroup \
    && adduser --system --uid 1000 --gid 1000 appuser

COPY --from=build /app .
USER appuser

EXPOSE 8080
ENV ASPNETCORE_URLS=http://+:8080
ENV ASPNETCORE_ENVIRONMENT=Production

ENTRYPOINT ["dotnet", "OpenCode.DragonBall.Api.dll"]
```

**src/OpenCode.Music.Api/Dockerfile:**

    -o /app \

RUN addgroup --system --gid 1000 appgroup \
    && adduser --system --uid 1000 --gid 1000 appuser

COPY --from=build /app .

EXPOSE 8080

ENV ASPNETCORE_ENVIRONMENT=Production

ENTRYPOINT ["dotnet", "OpenCode.Music.Api.dll"]

**Design decisions:**

1. **Copy Directory.Packages.props and OpenCode.slnx** for NuGet version centralization вЂ” the .NET project system reads these for package version resolution. Without them, `dotnet restore` fails because it can't find `$(Directory.Packages.props)`.

2. **Separate csproj restore from source copy** вЂ” Docker layer caching: project files change less often than source files. `dotnet restore` only reruns when csproj or Directory.Packages.props change.

3. **Runtime stage uses `addgroup`/`adduser`** вЂ” the `mcr.microsoft.com/dotnet/aspnet:10.0` image is Ubuntu-based, so `adduser --disabled-password` doesn't work (that's Alpine syntax). Use:
   ```dockerfile
   RUN addgroup --system --gid 1000 appgroup \
       && adduser --system --uid 1000 --gid 1000 appuser
   ```

4. **--no-restore on publish** вЂ” restore already ran in an earlier layer; `--no-restore` prevents unnecessary network calls.

5. **Build context is the project root** (`.` in docker-compose.yml) because the .NET projects reference each other and need the solution structure.

**src/OpenCode.Frontend/nginx.conf:**

```nginx
server {
    listen       80;
    server_name  localhost;
    root   /usr/share/nginx/html;
    index  index.html;

    # Gzip compression for better performance
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml text/javascript image/svg+xml;
    gzip_min_length 256;

    # API requests proxy to Kong (optional вЂ” frontend calls Kong directly from browser)
    # This is a reverse-proxy fallback if needed, but primary communication is browserв†’Kong directly.
    # Uncomment if you want to hide Kong from the browser:
    # location /api/ {
    #     proxy_pass http://Kong:9080;
    #     proxy_set_header Host $host;
    #     proxy_set_header X-Real-IP $remote_addr;
    # }

    # SPA fallback вЂ” all non-file routes serve index.html
    location / {
        try_files $uri $uri/ /index.html;
        expires -1;
        add_header Cache-Control "no-store, no-cache, must-revalidate";
    }

    # Static assets with long cache
    location /assets/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # Security headers
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-Frame-Options "DENY" always;
    add_header X-XSS-Protection "1; mode=block" always;

    # Error pages
    error_page 404 /index.html;
    error_page 500 502 503 504 /index.html;
}
```

**src/OpenCode.Frontend/Dockerfile:**

```dockerfile
# Stage 1: Build the Vite SPA
FROM node:20-alpine AS build
WORKDIR /app

# Copy dependency manifests and install (layer cache)
COPY package.json package-lock.json ./
RUN npm ci

# Copy source and build
COPY . .
RUN npm run build

# Stage 2: Serve with Nginx
FROM nginx:alpine AS runtime

# Copy nginx config
COPY nginx.conf /etc/nginx/conf.d/default.conf

# Copy built assets from build stage
COPY --from=build /app/dist /usr/share/nginx/html

# Create non-root user and switch
RUN adduser -D -u 1000 -g 1000 -s /bin/sh nginxuser \
    && chown -R nginxuser:nginxuser /usr/share/nginx/html \
    && chown -R nginxuser:nginxuser /var/cache/nginx \
    && chown -R nginxuser:nginxuser /var/log/nginx \
    && chown -R nginxuser:nginxuser /etc/nginx/conf.d \
    && touch /var/run/nginx.pid \
    && chown -R nginxuser:nginxuser /var/run/nginx.pid

USER nginxuser

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

**Design decisions:**

1. **`node:20-alpine` for build** вЂ” small image, fast npm ci, official Node image (no Bitnami). Alpine avoids unnecessary tooling.

2. **`nginx:alpine` for runtime** вЂ” official Nginx image, minimal surface area. The default nginx:alpine config is replaced by our custom nginx.conf.

3. **`npm ci` instead of `npm install`** вЂ” uses `package-lock.json` for deterministic builds, fails if lockfile is out of sync. This is the production-correct choice.

4. **SPA routing** вЂ” `try_files $uri $uri/ /index.html;` ensures React Router works on all paths (refresh on `/dragonball/characters` serves index.html, not a 404).

5. **Non-root user in Alpine** вЂ” `adduser -D` creates a system user without password. Alpine uses busybox's adduser, which has different syntax than Debian/Ubuntu's adduser. The nginx:alpine image already has a `nginx` user; use that instead of creating a new one:

   Actually, the `nginx:alpine` image already has an `nginx` user defined. Use:
   ```dockerfile
   USER nginx
   ```
   This is simpler and safer.

   **Correction:** The nginx:alpine image already runs as the `nginx` user by default when you use the `nginx -g 'daemon off;'` CMD. However, we need to ensure file permissions are correct:
   ```dockerfile
   RUN chown -R nginx:nginx /usr/share/nginx/html
   USER nginx
   ```

   **Final approach:** Use the built-in `nginx` user from the nginx:alpine image:
   ```dockerfile
   RUN chown -R nginx:nginx /usr/share/nginx/html
   USER nginx
   ```

6. **Security headers** in nginx.conf: X-Content-Type-Options, X-Frame-Options, X-XSS-Protection.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Container в†’ Container | Internal Docker network вЂ” trusted traffic between containers |
| Host в†’ Docker socket | Not mounted вЂ” containers cannot access host Docker daemon |
| Container runtime | Non-root user limits blast radius of container compromise |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
