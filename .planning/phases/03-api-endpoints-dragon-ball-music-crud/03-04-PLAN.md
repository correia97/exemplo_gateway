  <action>
    Register a document transformer that overrides the OpenAPI server URL to match the APISIX upstream path.

    **Using an inline lambda approach** (no separate class file needed):

    A) **`src/OpenCode.DragonBall.Api/Program.cs`** — Modify the `AddOpenApi()` registration to include a document transformer:

    Replace:
    ```csharp
    builder.Services.AddOpenApi();
    ```
    With:
    ```csharp
    builder.Services.AddOpenApi(options =>
    {
        options.AddDocumentTransformer((document, context, cancellationToken) =>
        {
            document.Servers = [new OpenApiServer { Url = "/api/dragonball" }];
            return Task.CompletedTask;
        });
    });
    ```

    Add the required using directive at the top of Program.cs:
    ```csharp
    using Microsoft.OpenApi.Models;
    ```

    B) **`src/OpenCode.Music.Api/Program.cs`** — Same pattern, different server URL:

    Replace:
    ```csharp
    builder.Services.AddOpenApi();
    ```
    With:
    ```csharp
    builder.Services.AddOpenApi(options =>
    {
        options.AddDocumentTransformer((document, context, cancellationToken) =>
        {
            document.Servers = [new OpenApiServer { Url = "/api/music" }];
            return Task.CompletedTask;
        });
    });
    ```

    Add the required using directive at the top of Program.cs:
    ```csharp
    using Microsoft.OpenApi.Models;
    ```

    **Rationale for inline lambda over separate class:**
    - The transformer is a single line of logic — no benefit from a separate file
    - Keeps the configuration co-located with the service registration
    - Avoids scattered one-off classes in the project
    - The `Microsoft.OpenApi.Models.OpenApiServer` and `OpenApiDocument.Servers` types handle the spec correctly

    **What this achieves:** When Scalar loads the OpenAPI document, it reads the server URL from the spec. By overriding it to `/api/dragonball` (or `/api/music`), all API requests in the Scalar UI will be prefixed with the APISIX upstream path instead of the raw `localhost:{port}`. This is required by DBALL-09 and MUSIC-14.
